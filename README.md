## ğŸ‡ºğŸ‡¸ English

A standalone, zero-dependency JavaScript library that parses binary font files (TTF / OTF / WOFF) and converts them into Three.js-compatible typeface JSON â€” entirely in the browser. Includes a custom Shape builder that bypasses Three.js's broken winding detection for reliable rendering of complex glyphs.

### âœ¨ Why This Library?

Three.js's built-in `TextGeometry` requires fonts in a proprietary JSON format, typically generated by the now-deprecated [Facetype.js](https://gero3.github.io/facetype.js/) converter. This creates friction:

- Users must convert fonts manually via an external tool.
- The converter doesn't handle CFF/OTF fonts reliably.
- Three.js's `ShapePath.toShapes()` uses only 12 divisions for area calculation, causing **winding direction detection to fail** on complex CFF glyphs â€” resulting in holes rendered as solid fills, or missing shapes entirely.

**3D Font Engine** solves all of these problems:

| Problem | Solution |
|---|---|
| Manual JSON conversion required | Direct .ttf / .otf / .woff parsing in the browser |
| CFF fonts fail to render | Full CFF Type2 CharString interpreter |
| Complex glyphs have holes/fills inverted | Custom Shape builder with proper winding & point-in-polygon testing |
| No kerning support | kern table + GPOS pair positioning support |
| External dependencies needed | Zero dependencies (inflate decoder built-in) |

### ğŸ”§ Features

**Font Parsing**
- TrueType `glyf` table with simple & composite glyph support
- Full CFF Type2 CharString interpreter (shared stack, subroutines, transient array)
- CFF2 parsing with blend operator support (default instance)
- CID-keyed font support (FDSelect, per-FD Private DICTs)
- `cmap` formats 0, 4, 6, 12

**Kerning**
- Legacy `kern` table (format 0)
- GPOS pair positioning (format 1 & 2, including class-based)

**Container**
- WOFF decompression with built-in RFC 1951 inflate decoder
- WOFF2 detection with user-friendly error

**Three.js Integration**
- Outputs standard Three.js typeface JSON (compatible with `THREE.Font`)
- Custom `createTextShapes()` builder â€” bypasses `TextGeometry` winding bugs
- Proper area-based winding detection with point-in-polygon hole assignment
- Direct `ExtrudeGeometry` compatible output

**Utilities**
- SVG path generation for debugging and 2D preview
- Glyph-level SVG path export
